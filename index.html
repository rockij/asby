  • 텍스트간격
    • 행간(줄 사이 간격)은 글자 크기(높이)의 1.5배 이상
    • 단락 간 간격은 글자 크기의 2배 이상으로 간격(마진)을 두어야함
    • 자간(글자 사이 간격)을 글자 크기 .12배 이상
    • 어간(단어 사이 간격)은 글자 크기 .16배 이상
    • 예외사항: PDF, 이미지텍스트, 로고, 비디오캡션, canvas요소
  • 마우스 호버시에 적용되는 컨텐츠는 포커스시에도 적용이 되어야 한다 참고
  • 시간제한 알림은 시간연장 버튼을 추가하고, aria-live을 이용해 안내제공 참고
  • 인터랙션 애니메이션
    • 정지 추가 참고
    • prefers-reduced-motion: reduce (css media) (only firefox, safari) 를 통해 정적인 이미지 제공 가능
  • 동시 입력 매커니즘 고려 (키보드접근, 문자단추키, 레이블과 접근 가능한 이름)
  • aria-lable
    • 이미지나 아이콘등 숨김 텍스트가 필요한 곳에 대신 사용
    • p, i, u, del, span, div..에 텍스트가 있다면 우선순위로 텍스트를 읽는다
    • 문구는 간력하게 하고 길어야할 경우 aria-describedby 사용
  • aria-live 활용하여 사용자를 고려한 상태메시지 참고
  • datepicker 적용 (ex:대한항공) 참고
  • 별점표시 '별다섯개중 몇개'
  • 랜드마크
    • 랜드마크에 대해서 연결
    • baner, main, complementary, contentinfo 최상위 랜드마크
    • 예를들어 main에 baner랜드마크가 들어가있다면 잘못된 랜드마크
    • 여러개의 랜드마크 사용시 aria-label, aria-labelledby 사용
    • div를 사용한다면 role을 사용
    • banner는 페이지에 하나만 존재, sectioning 콘텐츠로는 header와 동일
    • complementary는 페이지에 하나만 존재, sectioning 콘텐츠로는 aside와 동일
    • contentinfo는 페이지에 하나만 존재, sectioning 콘텐츠로는 footer와 동일
    • form은 search랜드마그가 아닌 영역, 레이블 제공(legend)
    • search랜드마크는 aria를 사용해서만 부여해줄 수 있다 role="search"
    • main은 페이지에 하나만 존재 메인컨텐츠에 적용
    • region 랜드마크는 section과 동일
  • title 속성은 툴팁에 적절할 것 같은 조언 정보를 나타내므로 접근성 측면에선 aria-label 권장
  • 윈도우 고대비 모드 Shift+Alt+PrtSc, 색에 무관한 콘텐츠 인식 참고
  • 리스트 형태의 많은 li가 있더라도 하나의 ul안에 들어가도록 한다
  • 목록스타일을 제거하면(list-style:none) iOS에서 목록 의미가 제거됨 참고
